    A. 场景与触发
    先讲清楚这是什么场景、什么时候触发、最终想要什么结果。
    0/6
    建议用“对象 + 动作 + 结果”命名，例如：会议纪要与行动项提炼。
    描述真实使用时机，不要写岗位背景。
    一行一句常见说法，后续会参与触发策略和 Prompt 生成。
    可以配置自动触发、关键词触发、Slash Command 或工具直达触发。
    用自然语言描述什么情况下应该触发该 Skill。
    如果只允许显式调用，最终 Prompt 和命令建议里会体现这一点。
    只写最终交付物，不写过程。
    B. 输入、输出与流程
    把输入类型、输出要求、验收标准和流程步骤写清楚。
    0/8
    这里写输入里通常会有什么，而不是具体哪一份文件。
    可以勾选预置标准，再补充专属标准。

    每条规则都按“如果……那么……”来写。
    C. 约束与参考
    把模板、参考角色、约束、参考材料和补充背景集中到最后一步。
    0/6
    如果你已经有固定模板或强参考，请尽量写出来。
    推荐字数范围：50 到 200 字。
